# Codex 작업지시서: 플레이테스트 판정 — "매력적인가?" (경기 관전 제외)

작성일: 2026-07-07
방법: 실제 유저처럼 6일 플레이(2026-04-01~04-06, LG): 데스크 확인→결재→경기 스킵→진행 반복, 시장/기록실 방문. 모든 상태 전이를 로그로 기록.

## 0. 판정 요약

**뼈대는 매력 있다.** 아침 데스크→결재→오늘 경기→내일로 이어지는 리듬, 실명 KBO 로스터, 결재 콘텐츠(콜업/트레이드/부상), 서사 메모리 — "내 구단의 하루를 산다"는 감각의 골격은 진짜다. 그러나 **6일 플레이에서 몰입을 끊는 순간이 5번 왔고, 전부 수리 가능한 UX/루프 문제다.** 지금의 최대 적은 기능 부족이 아니라 마찰이다.

## 1. 실제로 겪은 마찰 (발생 순서대로)

### F1. blocking 결재가 "벽"인데 처리할 버튼이 안 보인다 (최악의 순간)
- 4/6 문정빈 부상 결재가 열리자 진행이 막혔고 status에 "먼저 처리해야 날짜를 넘길 수 있습니다"만 반복됐다. **현재 탭(순위)에는 결재 버튼이 없어** 같은 자리에서 6번 막혔다. 유저는 "어딜 눌러야?"를 스스로 찾아 뉴스함으로 가야 한다.
- **수리**: blocking 결재는 어느 탭에서든 **즉시 뜨는 모달**(또는 전 탭 상단 고정 패널)로 — 옵션 버튼 포함. 막히는 순간 = 처리하는 순간이어야 한다.

### F2. 결정이 조용히 썩는다 — 만료가 무언극이다
- 시작 시점의 흥미로운 결재 2건(김주성 콜업, SSG 트레이드 제안)이 있었는데, 진행 중 어느 순간 **소리 없이 사라졌다**(만료 자동처리). 결정은 이 게임의 핵심 콘텐츠인데 소비를 유도하지 못하고 증발했다.
- **수리**: ① 만료 D-1 리마인드 메일("내일 자동 패스됩니다"), ② 만료 처리 시 데스크/status에 명시 통보("SSG 제안이 기한 만료로 거절 처리됨"), ③ [계속] 멈춤 사유에 "만료 임박 결재" 추가.

### F3. 같은 잔소리 메일이 매일 온다
- "[운영팀] 외국인 선수 등록 인원 초과 점검"이 **매일 새 메일**로 도착(3일 연속 최상단). 상태가 그대로면 조건이 계속 참이라 매일 발송 — 인박스 신뢰를 깎는다.
- **수리**: 상태 지속형 경고는 1회 발송 + 해소/악화 시에만 재발송(마일스톤 ledger처럼 발송 기록 dedupe).

### F4. unread 폭주 지속 (기존 WP-H 미적용 확인)
- 6일 만에 128→160통. [모두 읽음]/7일 자동 읽음/뱃지 99+ 캡/데스크 "새 편지"는 오늘분만 — WP-H 그대로 시행할 것.

### F5. 승패에 무게가 없다
- 2승5패→3승5패로 미끄러졌는데 세계는 심드렁했다(메일 몇 통). 패배의 공포도 승리의 보상도 없음 — **FM-1(구단주 기대·신뢰·경질)이 정확히 이 구멍의 답**이다. 우선순위 재확인. 소형 보완: 데스크에 "구단주 심기/기대 순위 대비" 한 줄이라도 먼저(FM-1 선행 티저).

### F6. 경기 결과의 감정 비트가 status 한 줄
- "경기 스킵 완료: 한화 9-6 LG"가 전부. 스킵 유저(대다수)에게 승/패 감정 비트가 없다.
- **수리**: 스킵 직후 **결과 카드 팝**(스코어+결승 장면 한 줄+수훈 선수, 승리 시 팀컬러/패배 시 무채색, 1.5초) + WP-F 박스스코어 모달 연결. 5연패 중이면 카드에 그 무게 반영.

### F7. 스킵 루프가 메일/결재를 우회한다
- "경기 스킵"만 연타해도 하루가 돌아가서 인박스를 영영 안 보게 된다(내가 그랬다).
- **수리**: 경기 스킵 직후 open 결재가 있으면 결과 카드에 "결재 N건 대기" 버튼 동봉. 결재함이 비어야 [계속]이 조용히 지나가게.

## 2. 우선순위

```
F1(blocking 모달) → F2(만료 예고/통보) → F3(잔소리 dedupe) → F4(WP-H) → F6(결과 카드) → F7(스킵 리마인드) → F5(FM-1로)
```
전부 UI/메일 계층 — 엔진·밸런스 무관, verify 3종 유지. F1~F4는 하루면 된다. 이 7개가 끝나면 "매력 있는 뼈대"가 아니라 그냥 "매력 있는 게임"이 된다.
