# Codex 작업지시서: FM 시스템 이식 — KBO 야구식

작성일: 2026-07-07
선행: `docs/codex-season-core-work-order.md`(WP-A~I — **계속 최우선**). 이 문서는 그 다음 사이클: Football Manager를 FM답게 만드는 시스템들을 KBO 감독/GM 하이브리드에 맞게 이식하는 로드맵이다.

## 0. 검증 결과 (2026-07-07 시점)

- `verify` 34항목 전부 PASS. **WP-A(기록실+커리어 히스토리)가 작업트리에 구현 완료**(미커밋): `state.leagueHistory`, `player.history` 보존 verify 신규 PASS, "기록실" 탭(리그 리더보드/역대 시즌) UI 확인. → **커밋부터 할 것.**
- WP-B(드래프트 유저 지명)~WP-I: 미착수. 시즌 코어 지시서 순서대로 계속.

## 1. 왜 이 시스템들인가

FM의 중독성은 경기 시뮬이 아니라 **"나"가 세계 안에 존재한다는 압력**에서 온다: 구단주가 나를 평가하고, 실패하면 잘리고, 선수가 나에게 불만을 말하고, 기자가 내 말을 문제 삼는다. 현재 게임은 하루 루프·결재·서사까지 갖췄지만 **패배 조건과 인간 관계가 없다** — 아무리 져도 아무 일도 일어나지 않는다. 아래 FM-1이 가장 먼저인 이유다.

## 2. 작업 패키지 (FM 시리즈)

### FM-1. 구단주 기대치 · 신뢰도 · 경질 (패배 조건 = 긴장감)
- **스펙**: ① 새 게임 시작 시 구단 전력지수로 **시즌 목표** 산출(우승 후보/가을야구/5할/리빌딩 — 온보딩 취임 기자회견에서 고지, 기존 appointment 화면 확장). ② `state.managerJob = { trust: 0~100, seasonGoal, evaluations[] }` — 신뢰도는 목표 대비 순위 페이스/연패/구단주 월례 평가(기존 owner-monthly 메일 재사용)로 증감. ③ trust 임계 이하 → 경고 메일 → **경질**(게임오버 화면: 커리어 요약 + 새 팀 제안 2~3개 or 종료). ④ 시즌 종료 시 목표 달성도에 따라 재계약/연봉 인상/타 구단 이직 제안 메일. ⑤ 감독 커리어 페이지(통산 성적/수상/역임 구단 — WP-A의 leagueHistory 재사용).
- **KBO식**: 목표 문구를 KBO 관례로("가을야구 진출", "5강 경쟁", "리빌딩 원년"). 경질 사유도 KBO식 보도문 톤.
- **수용**: 목표 미달+장기 연패 시뮬에서 경질 발생, 경질 후 새 팀 제안 수락 시 그 구단으로 이어서 플레이(selectedTeamId 전환+세이브 호환). verify: trust 증감/경질 트리거 케이스.

### FM-2. 코칭스태프 v1 (not-started 확인됨)
- **스펙**: 슬롯 6개 — 수석/투수/타격/배터리/수비주루/퓨처스 감독. **코드형 인물**(`COACH-###`, 실명 임의 생성 금지 원칙 유지) + 성향/능력치(투수육성·타자육성·컨디션관리·사기관리 1~20). 효과: 성장 공식 가중, 부상 발생률 보정, 팀 사기 회복 속도. 계약(연 단위)·영입/교체는 오프시즌만, 시장 풀 12~20명. 프런트 탭에 스태프 보드.
- **수용**: 투수코치 육성치 상위 팀의 투수 성장률이 유의미하게 높음(시즌 시뮬 비교), 스태프 저장 roundtrip.

### FM-3. 훈련 시스템 v1
- **스펙**: 선수별 **훈련 포커스 1슬롯**(타격/선구안/수비/주루/구속/제구/체력/회복) — 라인업 탭 or 선수 상세에서 지정, 기본값 자동(약점 보완). 효과: 기존 성장/노쇠 공식에서 포커스 능력치 성장 가중(+코치 보정), 과부하 조합(체력 낮은데 고강도)은 부상 위험 증가. 주간 훈련 리포트 메일(성장 조짐 TOP3 — 기존 주간 리뷰에 통합 가능). 퓨처스 방침(실전 위주/기초 위주) 팀 단위 토글.
- **수용**: 같은 선수 시드에서 포커스 유무로 시즌 말 능력치 차이 발생(결정론 유지 — 포커스가 시드 입력에 포함), balance PASS(경기 확률 자체는 무변경).

### FM-4. 선수 면담 · 불만 · 약속
- **스펙**: 기존 `player.morale`+개인성(archetype) 위에 ① **불만 상태**: 출전 부족(주전급인데 벤치), 2군 강등 불만, 연봉 불만(연봉협상 WP-I 연동) — 불만 발생 시 본인 발신 메일("면담 요청"). ② **면담 command**(결재 UI 재사용): 격려/질책/약속(출전 기회·1군 등록·연봉 반영). ③ **약속 원장**: `state.promises[]` — 기한 내 이행 여부 추적, 파기 시 사기 급락+서사 아크. ④ 주장/베테랑(나이·경력 상위)의 사기가 팀 사기에 전파 가중.
- **KBO식**: 면담 문구 톤(선후배 문화), 베테랑 예우 이슈(고참 2군행 = 파장 큼 — 이미 슬럼프 주전 결재 있음, 그 후속으로 연결).
- **수용**: 주전급 3주 벤치 → 불만 메일 → 약속 → 이행/파기 분기 동작. 결재 이력에 남고 저장 보존.

### FM-5. 스카우팅 안개(fog of war) v1
- **문제**: 스카우트 리포트 루프는 있으나 **타팀 선수 능력치가 그대로 노출** — FM의 정보 게임이 없다.
- **스펙**: ① 타팀 선수의 OVR/POT/세부 능력치를 기본적으로 **범위 표시**(예: OVR 120~150, 리포트 0건이면 넓게). ② 스카우트 지시 완료마다 해당 선수/팀 범위 축소(관측치는 `hashParts` 결정론 오차). ③ 트레이드/FA 화면도 관측치 기준(이 판단 리스크가 재미). ④ 자팀은 항상 정확. ⑤ 설정에 "정보 공개 모드"(기존 방식) 토글 — 라이트 유저 배려.
- **수용**: 리포트 0건 선수는 범위 표기, 3건 후 ±5 이내 수렴. verify: 관측치 결정론.

### FM-6. 기자회견 · 언론 관계 v1
- **스펙**: 기존 interview-request 결재 확장 — ① 시리즈 종료/위닝·루징 시리즈 후 기자회견 메일(질문 2~3개, 답변 3택). ② 답변 톤 → 팀 사기/특정 선수 사기/서사 아크에 소폭 반영(이미 있는 파이프). ③ 기자 관계도(매체별 우호도): 회피성 답변 반복 시 비판 기사 톤. 빈도 상한 주 2회.
- **수용**: 답변 선택이 다음 날 기사 톤에 반영, 같은 시드 같은 결과.

### FM-7. KBO 캘린더 디테일 (WP-D 완료 후)
- 스프링캠프지 선택(오키나와/애리조나/국내 — 캠프 효과 보정: 컨디션 상승 속도), 비활동기간(12/1~1/31 훈련 지시 불가), 시범경기(3월 중순 5~8경기 — campStats 분리 원칙 유지), 올스타전 이벤트(팬 투표 메일+당일 이벤트 기사).

## 3. 순서

```
[선행] 시즌 코어 WP-B~I 완료 (특히 WP-C 인터리브는 단독 사이클)
FM-1 (경질/목표 — 게임에 긴장감) → FM-2 (스태프) → FM-3 (훈련) → FM-4 (면담/약속)
→ FM-5 (스카우팅 안개) → FM-6 (기자회견) → FM-7 (캘린더 디테일)
```
- FM-1은 시즌 코어와 독립적이라 WP-C(엔진) 진행 중 병행 가능(파일 겹침 적음).
- 전 패키지 공통: 실명 임의 생성 금지(스태프/기자 = 코드형·역할명), `hashParts` 결정론, verify 3종+신규 항목, 저장 roundtrip/마이그레이션.
