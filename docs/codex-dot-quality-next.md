# Codex 지시서: 도트 퀄리티 다음 단계

작성일: 2026-07-07. 전제: 배치/경계/60fps는 검증 완료 — 이제 순수 "그림의 급"을 올리는 사이클.

## 우선순위 (효과 순)

### 1. v2 완주가 도트 퀄의 최대 레버 (R2~R6)
이미지젠 필드(잠실 주간 캡처 기준)는 이미 절차 드로잉과 급이 다르다. **구 렌더러의 절차 필드에 더 투자하지 말 것** — v2 마일스톤(수비 배치→투구→타구/주루→카메라/스킨→전환)을 끝내는 게 어떤 스프라이트 작업보다 화면 퀄을 올린다. 앵커 수정 4건(r1-review 문서) 선행.

### 2. 스프라이트 128px 고화질 재생성
- 64px → **128px 셀**(소스는 셀당 256px 생성 후 2배 정수 축소). 얼굴 표정/유니폼 주름/장비(배팅장갑·팔토시·포수 마스크·미트) 디테일을 실제 경기 화면에서 식별할 수 있게 한다.
- 이미지젠 프롬프트 고정 요소: "chibi pixel art baseball player, 1px dark outline, flat 3-tone shading, transparent background, sprite sheet grid" + 예약색 규칙(모자 #d23b3b 등 기존 스펙). 시트 후처리는 기존 빌더(양자화·레지스트레이션 검사) 재사용.

### 3. 프레임 밀도 (부드러움의 나머지 절반)
- run 4→**6프레임**, swing 8→**10프레임**(임팩트 전후 1장씩 — 임팩트 홀드감 강화), 투구 릴리스 직후 팔로우 1장 추가.
- 포즈 전환에 1프레임 브리지(stance↔run 등 급전환 시 중간 자세) — 아틀라스 +6~8칸.

### 4. 셰이딩/라이팅 규칙 승격 (빌더에서 기계 검증)
- 음영 2단 → **3단**(베이스/셰이드/하이라이트) + 야간 경기 한정 **림라이트 1px**(조명 방향 상단 — 필드 조명과 통일).
- 아웃라인을 순수 검정 대신 **selout**(주변색의 어두운 톤): 팔레트 검사에 아웃라인 색 규칙 추가.
- 그림자 타원을 포즈 연동(다이브 시 길게, 점프 시 작고 옅게).

### 5. 픽셀 파티클 마감
- 슬라이드 먼지·타구 임팩트 스파크·홈런 컨페티를 **2~3px 픽셀 파티클**로 통일(현 rect 이펙트 승격). 야간에는 파티클도 톤 다운.
- 능력 배지 패드(수비수 발밑 노란 사각) 축소 또는 호버 시만 — 지금은 피규어 받침대처럼 보임.

## 순서
`앵커 수정 → R2~R4(v2 선수/플레이) → 스프라이트 128px 소스 재생성(2·3·4 동시) → R5~R6 → 파티클(5)`
가드레일 동일: 팔레트 상한/레지스트레이션 빌드 검사, verify 3종, 버스트 캡처 눈검수.
