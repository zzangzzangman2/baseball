# Codex 작업지시서: 게임캐스트 QA 2차 — 겹침·어색함·로직 결함

작성일: 2026-07-07
방법: 헤드리스 CDP로 랩(`gamecast-lab.html`)을 구동해 7개 장면 스크린샷 검수(초반 타석/홈런/x4 진행/스킵/이닝 교대/모바일/고척 매치업) + 신규 코드(8ab85f1..HEAD) 로직 리뷰 + `verify:browser` PASS 확인.
전제: 이전 작업지시서들은 전부 적용·삭제됨(b599bcb). 이 문서가 현재 유일한 할 일 목록이다.

먼저 잘된 것: 800px 대형 캔버스, 관중석·파울폴·워닝트랙이 있는 구장, 60프레임 애니 아틀라스, 이닝 교대 홀드+[계속] 버튼, 일시정지/x0.5/타석 확인/사운드 토글 전부 동작 확인. 큰 골격은 완성됐고 아래는 그 위의 결함들이다.

## 1. 버그 (심각도순)

### B-1. 스킵이 이닝 교대 홀드에 막힌다 (치명)
- 증상: x4로 진행 → 1회초 종료 홀드 → **[스킵] 클릭해도 이닝 교대 화면 그대로**(스크린샷 04·05 동일 화면). 경기 끝으로 못 가고, 종료 요약 화면도 도달 불가.
- 함께 수리: **배속과 홀드의 상호작용 규칙** 정의 — ① 스킵은 모든 홀드를 무시하고 최종 상태로, ② x2 이상 배속에서는 자동 홀드(이닝 교대/승부처)를 건너뛰거나 1.2초 자동 해제(설정), ③ [계속] 버튼과 Space 모두 홀드 해제.
- 수용: 스킵 → 종료 화면 즉시. x4 방치 시 유저 입력 없이 경기 완주.

### B-2. 좌익수 자리에 스프라이트 2개 + 마운드에 투수 없음 (모든 장면에서 재현)
- 증상: LF 코너에 수비수 두 명이 겹쳐 서 있고, 마운드는 투구 순간이 아닌 때에도 항상 비어 있다(스크린샷 01/02/04/07 전부).
- 의심 원인: 수비 배열(`gamecastDefensiveAlignment` 순서 LF,CF,RF,SS,2B,3B,1B,P,C)과 좌표/프레임을 zip하는 곳의 인덱스 시프트 — P가 LF 좌표를 받았을 가능성. Phaser 경로(`drawStaticDefense`)와 Canvas 경로(`drawPixelFielders`) 둘 다 확인.
- 수용: 수비 9명이 각자 자리에 1명씩(투구 애니메이션 중 P 숨김 제외), 랩 3장면 스크린샷 재검.

### B-3. 정체불명 스프라이트 1개 (2루 베이스 뒤)
- 증상: 볼넷 장면(스크린샷 07)에서 2루 뒤(얕은 중견 방향)에 소속 불명의 회색 계열 스프라이트 1개. 주자도 수비수도 심판도 아닌 위치.
- B-2와 같은 매핑 버그의 다른 얼굴일 가능성 높음 — 함께 조사.

### B-4. 스텝 모드가 탭 전환 복귀 시 무시된다
- ui.js:5614 부근 — 가시성 복귀 시 `!state.paused`만 보고 `stepMode` 홀드 상태를 무시하고 `phaser.resume()`. 재현: 타석 확인 모드 → 탭 전환 → 복귀 → 자동 재생.
- 수리: 재개 조건에 stepMode 홀드 포함.

### B-5. 일시정지/홀드 정지 프레임마다 필드 per-pixel 재드로잉
- `paintStaticHoldFrame`이 매 정지 시 `drawFieldCanvas`(width×height 픽셀 루프)를 다시 실행(gamecastPhaser.js:505-530 부근). 이닝 교대 홀드가 경기당 17+회 발생하는 현 구조에서 리사이즈가 아닌데도 풀 재계산 — 이 리포의 금지 패턴.
- 수리: 필드를 오프스크린 캐시로 1회 굽고 정지 프레임은 `drawImage` 복사만.

### B-6. 타자/포수 depth 정렬 경계 케이스
- `compareGamecastActorDepth`(gamecastPhaser.js:813-834)의 역할별 y 오프셋(batter +2, umpire +4)이 동일 y에서 포수-타자 순서를 뒤집을 수 있음. 홈 클러스터(포수+타자+심판)가 가장 겹치는 지점이므로 역할 우선순위를 명시적 tiebreaker로.

### B-7. 빌드 스크립트 애니 메타 무검증
- `build_gamecast_sprites.py`의 V2_ANIMATIONS에서 frames/durations 길이 불일치를 빌드 시점에 잡지 않음 — 런타임에서만 조용히 어긋남. `validate_registration()`처럼 길이·프레임 존재 검증 추가, 불일치 시 빌드 실패.

## 2. 어색함/퀄리티 (체감 순)

### U-1. 구장 스킨이 사실상 동일하다
- 한화 vs 키움(고척 홈) 장면이 잠실 장면과 **완전히 같은 그림**(스크린샷 01 vs 07). 돔 천장/실내광 없음, 파크별 팔레트·펜스 차이 안 보임.
- 최소선: ① 고척 = 실내광(관중석 어둡게+천장 라인+조명 균일), ② 구장별 잔디 톤/담장색/좌석 팔레트 3~4종, ③ 전광판 위치·형태 1~2 변형. 랩 `?team=`으로 스크린샷 비교가 곧 수용 기준.

### U-2. "이닝 교대" 오버레이가 시스템 폰트
- 도트 세계 위에 매끈한 웹폰트 대형 텍스트가 떠서 이질적(스크린샷 04). "INNING BREAK" 배지도 동일. 픽셀 슬레이트(전광판풍 블록레터 — `blockLetters` 재활용)나 도트 패널로 교체. "계속 ▶" 버튼은 UI 계열이라 유지 OK.

### U-3. 전광판 이닝 표기 "1초"
- "1회초"의 축약 "1초"가 시간(second)처럼 읽힘. "1회초"/"1회말" 전체 표기 또는 "1▲/1▼" 관례 사용. HUD 좌상단·매치업 칩("1초 · 1OUT")도 동일 수정.

### U-4. 잔디 동심원 링 대비가 너무 강함
- 밝은 민트/진초록 링 교차가 어지럽고 시선을 뺏음(전 장면). 링 간 명도차를 절반 이하로 줄이거나 실제 구장풍 모잉 스트라이프(방사형 대신 평행선)로. 팀별 패턴은 U-1과 함께.

### U-5. 이닝 교대 때 수비수가 제자리에 서 있음
- 교대 홀드 화면에서 9명이 그대로 얼어 있음 — 더그아웃으로 뛰어 들어가고 새 수비가 나오는 러닝 연출(기존 run 프레임 재활용)이면 교대가 "보인다". 홀드 배경 연출로 최적.

### U-6. 사이드 피드가 "1회 초 삼성"만 반복
- 피드 리스트에 팀 라벨만 보이고 결과 문장이 없어 정보가 0(스크린샷 04 좌하단 4개 항목 동일 텍스트). 각 항목에 한 줄 결과문("구자욱, 좌월 솔로 홈런") 표시 — 문장 팩이 매치업 칩에는 적용됐으니 피드에도 연결.

### U-7. 모바일 버튼 세로 줄바꿈
- "정지"가 "정\n지", "타석 확인"이 "타석 확\n인"으로 꺾임(스크린샷 06). 버튼 `white-space: nowrap` + 최소폭/축약 라벨.

### U-8. 관중석 상단 검은 밴드 단조
- 상단 1/3이 큰 어두운 줄무늬 벽으로 비어 보임. 2층 관중 밀도 살짝 추가, 전광판 주변 구조물, 조명탑 정도로 밀도 보강(F-3 잔여). 우선순위 낮음.

### U-9. 타임라인(P-5) 미구현 리마인드
- 이닝 점프/다음 득점 장면 이동이 여전히 없음. B-1 스킵 수리와 같은 코드 영역이므로 함께 처리 권장.

## 3. 작업 순서 제안

```
B-1(스킵×홀드 규칙) → B-2/B-3(수비 배치 버그, 같은 원인 의심) → B-4/B-5(수명주기·성능)
→ U-1(구장 스킨)+U-4(잔디) → U-2/U-3(도트 폰트·표기) → U-5(교대 연출) → U-6/U-7 → B-6/B-7 → U-8/U-9
```

## 4. 검증

```powershell
npm run verify && npm run verify:balance && npm run verify:browser
```
- verify:browser 랩 검사에 추가할 것: ① 스킵 클릭 → 종료 상태 도달, ② 수비 스프라이트 좌표 중복 없음(9개 고유 위치), ③ x4 + 홀드 자동 해제 경로.
- 수리 후 랩 스크린샷 재촬영 비교(잠실/고척, 이닝 교대, 종료 화면). 스크린샷 스크립트는 세션 스크래치에 있던 것을 참고해 `tools/capture_gamecast_shots.mjs`로 리포에 넣어도 좋다(CDP 패턴은 verify_browser.mjs와 동일).
