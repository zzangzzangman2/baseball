# Codex 작업지시서: 게임캐스트 디자인 감사 — 단일 투영으로 기하를 통일하라

작성일: 2026-07-07
방법: PF 수리 후 1300px에서 420ms×16프레임 버스트 재검(홈런 PA + 내야 아웃 PA + 투구 장면).

## 0. PF 검증 결과

| 항목 | 판정 |
| --- | --- |
| PF-1 투구 페이즈 | **개선 확인** — 마운드 투수 + 릴리스 순간 공 + "공이 홈플레이트로 들어옵니다" 캡션 목격 |
| PF-3 결과 연기 | **미흡** — 홈런: 주자가 잠깐 나타났다 **3루 방향(왼쪽)으로 이동 후 소멸**, 라운딩 없음. 내야 아웃: 수비수 전원 정지 유지 |

## 1. 디자인 총평 — "위치가 이상해 보이는" 근본 원인은 좌표계 3개가 싸우고 있어서다

새 TV 중계 앵글 리디자인은 방향이 맞다. 그러나 지금 화면엔 **서로 다른 투영 3개가 공존**한다: ① 외야 잔디/파울라인은 원근(위로 갈수록 퍼짐), ② 내야 다이아몬드는 무원근 평면 마름모, ③ 스프라이트는 모든 깊이에서 동일 크기(중견수=포수 크기). 손으로 찍은 좌표들이 서로 다른 계에 속해 있으니 **무엇을 고쳐도 계속 어긋나 보인다.** 개별 좌표 수정이 아니라 투영 통일이 답이다.

## 2. 처방 D-1 (최우선): 단일 투영 함수

- `projectField(fx, fy) -> {x, y, depthScale}` 하나를 정의하고(필드 좌표: 홈=원점, 1루선=+x, 3루선=-x, 외야=+y), **모든 것**을 이 함수에서 도출: 베이스 4개, 마운드, 배터박스/홈플레이트, 파울라인(홈→폴), 파울폴(담장 교차점), 수비 9포지션, 주자 경로(홈→1→2→3→홈), 타구 궤적, 담장 곡선.
- 스프라이트 `depthScale` 적용: 홈 1.0 → 외야 0.72~0.8 (픽셀아트 유지 위해 3~4단 계단 스케일 허용).
- 손 좌표 상수 전면 제거. 목업(`docs/gamecast-quality-mockup.html`)의 검증된 기하를 투영 함수의 기준으로.
- **수용**: 베이스 마커가 베이스패스 교차점에 정확히 안착(현재 2루가 다이아몬드 꼭짓점 위에 부유), 파울폴이 담장 코너에 위치(현재 잔디 위 부유), 파울라인이 담장에서 종료(현재 관중석 관통), 주자가 홈→**1루(우측)** 방향으로 달림(현재 좌측 3루 방향 — 좌우 반전 버그).

## 3. 처방 D-2: 내야 형태를 야구장답게

- 통짜 주황 사각형 다이아몬드(현재 필드의 절반 크기) → 실제 내야: **베이스패스 띠 + 내야 잔디 컷 + 원형 마운드 + 홈플레이트 흙 원**. 크기도 축소(외야:내야 비율 현실화).
- 배터박스: 현재 포수보다 3배 큰 빈 브래킷 2개 → 스프라이트 스케일에 맞는 박스 + **타자는 박스 안에** 선다(현재 박스 밖 우측에 섬). 홈플레이트 오각형 추가.
- 마운드 사각 패치 → 원형 + 러버.

## 4. 처방 D-3: 아트 마감

1. 잔디: 통짜 민트 단색 → 모잉 스트라이프 2톤(명도차 6~10%로 은은하게 — 이전 동심원처럼 강하게 금지).
2. 내야 흙: 고채도 주황 단색 → 채도 낮춘 2톤 + 가장자리 1px 어두운 라인.
3. **대형 전광판이 텅 비어 있음**(노란 줄 하나) → 이닝/스코어/타자명 블록레터 렌더(기존 blockLetters 재사용). 못 채우면 구조물 자체를 축소.
4. HUD 카운트 핍이 점보트론과 **불일치**(B핍 2개 켜짐 vs 점보 "BALL 0-0") — 카운트 소스 단일화.
5. 관중석 상단 좌우 코너 빈 벽 → 조명탑/출입구 실루엣으로 마감.

## 5. 처방 D-4: 결과 연기 완결 (PF-3 재지시, 구체화)

- **홈런**: 타구를 밝은 트레일+외곽선으로 어두운 관중 배경에서도 보이게(글로우 1px) → 담장 통과 플래시 → **주자 4베이스 라운딩(투영 경로, 최소 1.2s)** → 홈인 후 점수/배지. 주자 중도 소멸 금지.
- **내야 아웃**: 담당 내야수 2~3걸음 이동 → 포구 → 1루 송구 라인+공 → 1루수 포구 → 배지. (수비수 "전원 정지" 프레임이 3연속 발견되면 실패)
- **수용**: 버스트 16프레임에서 홈런 주자 라운딩 ≥2프레임, 아웃 수비 이동 ≥2프레임, 타구 ≥2프레임.

## 6. 순서와 검증

D-1(투영 통일) → D-2(내야) → D-4(연기) → D-3(마감). D-1 없이 D-2/D-4를 하면 또 어긋난다 — 반드시 투영부터.

```powershell
npm run verify && npm run verify:browser && node tools/capture_gamecast_shots.mjs burst
```
체크리스트: ①주자 1루 방향 ②베이스 4개 안착 ③타자 박스 안 ④폴 담장 코너 ⑤타구 보임 ⑥주자 라운딩 ⑦수비 연기 ⑧핍=점보 카운트 일치. 가드레일 동일(결과 로직 무변경·결정론·reduced-motion·기존 P/홀드 페이스 유지).
