# Codex 작업지시서: 시즌 코어 — 경기진행·유저 결정권·기록실 (전면 점검 결과)

작성일: 2026-07-07
방법: 전체 기능 인벤토리(코드 확인, 함수:라인 근거) + 헤드리스 CDP로 전 탭 스크린샷 투어 + `verify` 33항목 확인.
전제: 게임캐스트/데일리 루프 지시서는 적용·삭제됨. **이 문서가 다음 사이클의 전체 목록이다.**

## 0. 현재 위치 (잘된 것 — 손대지 말 것)

화면 투어로 확인: 오늘의 데스크(일정/편지/어제 결과/성적·예산·관중 카드/다음 경기/전략 프리셋), 받은편지함 v2(읽음/결재함 필터/2컬럼/만료 자동처리), [계속 ▶ N결재] 멈춤 규칙, 정기+이벤트 메일 39종, 장기 서사(LG 메모리 heat 카드), 스카우트 지시→리포트 루프, 수동 타순/투수 운용, 게임캐스트(60프레임/페이싱/홀드). **FM식 하루 루프는 완성 단계다.** 남은 것은 그 하루가 굴리는 "시즌과 야구 자체"의 깊이다.

## 2. 작업 패키지 (우선순위순)

### WP-A. 기록실 + 커리어 히스토리 (최우선 — 지금 놓치면 세이브가 늙는다)
- **문제**: 시즌 롤오버가 `seasonStats`를 리셋만 하고 어디에도 보존 안 함(engine.js:2851-2861). 2027년이 되면 2026년의 모든 기록·수상·순위가 소멸 — 다년 플레이의 재미 원천이 없다.
- **스펙**: ① 롤오버 직전 `player.history[]`에 `{year, teamId, batting, pitching, fielding, awards}` push, `state.leagueHistory[]`에 `{year, standings, champion, awards, leaders}` push. ② 새 "기록실" 탭(또는 순위 탭 확장): 시즌 리더보드 — 타자 AVG/HR/RBI/SB/OPS, 투수 ERA/W/SV/HLD/K, **규정 필터**(타석=팀경기×3.1, 이닝=팀경기×1.0) + 미달 토글. ③ 팀 기록 테이블(팀 타율/ERA/득실점). ④ 선수 상세에 연도별 커리어 테이블. ⑤ 역대 시즌 요약(우승/MVP 목록).
- **수용**: 2026 완주→2027 롤오버 후 2026 기록/수상 전부 조회 가능. verify에 "롤오버 히스토리 보존" 추가. 저장 roundtrip PASS.

### WP-B. 드래프트/2차 드래프트 유저 직접 지명
- **스펙**: 픽 단위 진행으로 분리 — `advanceDraftPick(state)`(CPU 1픽), `commitUserDraftPick(state,{prospectId})`(검증: 중복지명/차례아님/무효id reject). `draft.pendingUserPick` 상태에서 자동 진행 정지(기존 blocking 결재 멈춤 패턴 재사용). UI: 후보 보드에 20-80 등급/포지션/나이 표시+필터, [지명] 버튼, "내 차례까지 진행"/"전체 자동(현행)" 둘 다. 2차 드래프트: `setSecondaryDraftProtection(state,{playerIds})`(35명 검증) — 자동 추천을 기본값으로 스왑 편집 UI, 내 픽 차례 직접 지명. 기존 `draft.rosterLedger`/`transferLedger` 흐름 유지(오프시즌 호환).
- **수용**: 11라운드 수동 완주 가능 + 전체 자동 결과는 현행과 동일. 보호명단 1명 스왑이 지명 풀에 반영. 기존 드래프트 verify PASS 유지 + 유저픽 reject 케이스 추가.

### WP-C. 경기 엔진 인터리브 (끝내기 + 연장 + 리버리지 불펜) — 별도 커밋 시리즈
- **문제**: 양팀 공격을 통째로 따로 돌려서(engine.js:3520-3543) 끝내기가 없고, 홈팀이 이기고 있어도 9회말을 다 치며, 마무리가 점수 상황을 모르고, 연장이 불가능하다. "경기진행"의 마지막 비현실.
- **스펙**: `simulateGame`을 하프이닝 인터리브 루프로 — `for inning { away half; if (!(inning>=9 && home 리드)) home half; walk-off 시 즉시 종료; }` + 동점 시 연장(정규 12회까지, 이후 무승부 / 포스트시즌 15회, `leaguePoliciesByDate` 데이터로). 타석 모델(`resolvePlateAppearance`)·교체 로직 재사용, 시드는 하프이닝 파생(`hashParts(gameSeed, inning, side)`). 불펜 선택에 실제 점수차/이닝 입력(세이브 상황 마무리). 기존 `forcePostseasonDecision` 제거.
- **리스크·게이트**: 9회말 스킵/끝내기로 리그 총 타석이 2~4% 줄어 **verify:balance 재튜닝이 한 세트**(AVG .278/ERA 4.64/HR 1.93 재통과까지 확률 계수 조정). 이것 때문에 다른 WP와 절대 섞지 말 것.
- **수용**: 시즌당 끝내기 승 20~60회, 무승부 리그 10~25회, 연장 경기 존재(라인스코어 10회+ 표시 — 게임캐스트/박스스코어 UI 포함), balance PASS, 기존 verify 전부 PASS. ※ 완료 후 게임캐스트 이닝 홀드가 "9회말 상황"을 알게 되어 P-3 홀드가 진짜 개입 창(Stage C)으로 승격 가능해진다.

### WP-D. 캘린더 v2 — 시리즈 구조
- **스펙**: 시즌 시작 시 720경기를 `CalendarGame{gameId,date,home,away,seriesId,status}`로 선생성 — 상대별 16경기(홈8/원정8), 화~목+금~일 3연전 축, 월요일 휴식, 개막 2연전, 올스타 브레이크(7/10~7/15 무경기). `simulateDay`는 캘린더 조회로 전환(생성기는 시드 고정 deterministic). 일정 탭에 시리즈 단위 표시. 기존 "시리즈 프리뷰" 메일의 상대-바뀜 휴리스틱을 seriesId 기반으로 교체.
- **수용**: verify invariant — 팀당 144, 상대별 16(홈8/원정8), 총 720, 팀당 하루 1경기, 월요일·올스타 기간 0경기.

### WP-E. 로스터 운영 완성 (엔트리 + 이동 UI + 방출 + 외국인 슬롯)
- **스펙**: ① `leaguePoliciesByDate`에 등록29/출장28(8/25부터 34/32), `registerPlayer`/`optionPlayer` command(말소 후 10일 재등록 제한 — 재등록 안내 메일은 이미 있음), 선수단 탭에 1군/퓨처스 구분+[등록]/[말소] 버튼+정원 카운터. 기존 futures-callup 결재를 이 command에 연결. ② `releasePlayer` command → 웨이버 공시 7일(기존 waiver-claim 메일 재사용) → 자유계약. ③ `player.foreignClass` + `validateForeignSlots`(보유 3+1) — 계약/트레이드/웨이버 게이트에 추가. ④ 라인업 검증이 등록 명단만 허용.
- **수용**: 정원 초과 등록 reject, 10일 내 재등록 reject, 외국인 4명째 일반 슬롯 reject, 유저 방출→클레임/FA 전환이 이벤트로그 추적. verify 케이스 일괄 추가.

### WP-F. 상세 박스스코어 모달 (반나절급 — 데이터는 이미 있다)
- 경기 카드 클릭 → 라인스코어(연장 칸 포함)/타자 전원 AB R H RBI BB SO/투수 IP H R ER BB K 투구수/결승타/이닝별 PA 로그. `boxScore`+`plateAppearanceEvents` 재사용, 엔진 무변경. 포스트시즌 경기에도 연결.

### WP-G. 재정 정합 + 수입 루프 v1
- **문제**: 수입이 없어 전 구단이 구조적 적자 — LG 개막 운영여력 -56억, 서사 1위가 영구히 "예산 압박". 예산이 압박이 아니라 소음이 됐다.
- **스펙**: 경기별 티켓수입(관중×객단가 — 관중수 이미 있음), 월 정산 메일(급여정산 메일 기존)과 연동해 수입/지출 양쪽 ledger, 모기업 지원+스폰서 고정수입(구단별 시드). 개막 시점 운영여력이 0~+30억 범위가 되도록 초기값 캘리브레이션. 예산 잔액을 트레이드/FA AI 상한에 반영.
- **수용**: 개막일 10개 구단 운영여력 전부 흑자~소폭, 시즌 수지가 ledger 합계와 일치, "예산 압박" 서사가 실제 과지출 팀에만 발생.

### WP-H. 뉴스함 위생 (소형)
- unread 128통 무한 누적 수리: ① [모두 읽음] 버튼, ② 7일 지난 비결재 메일 자동 읽음 처리, ③ 뱃지는 99+ 캡, ④ 데스크의 "새 편지 N통"은 **오늘 도착분만** 카운트. 결재는 예외(만료 처리 유지).

### WP-I. 연봉협상 v1 (스토브리그)
- 오프시즌 단계에 비FA 연봉 갱신: 성적/서비스타임 기반 제시액 자동 산출, 유저팀은 선수별 승인/조정(±20% 슬라이더) UI, CPU 자동. 거절 리스크는 v1에서 morale 하락으로 단순화. financeLedger 반영.

## 3. 순서와 병행 규칙

```
WP-A(기록실) → WP-B(드래프트 지명) → WP-F(박스스코어)   ← UI/데이터 트랙, 밸런스 무관
WP-C(인터리브 엔진)                                      ← 단독 커밋 시리즈, balance 재튜닝 세트
WP-D(캘린더) → WP-E(로스터 운영) → WP-G(재정) → WP-H/I
```
- WP-C만 경기 확률에 손댄다 — 다른 WP와 커밋을 섞지 말 것.
- 각 WP는 verify 3종 PASS + 신규 검증 항목 동봉(이 리포 관례).
- 게임캐스트 파일은 이번 사이클에서 손대지 않는다(WP-C의 연장 라인스코어 표시 제외).
