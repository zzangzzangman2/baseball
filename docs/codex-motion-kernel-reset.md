# Codex 설계도: 모션 커널 리셋 — "하나씩, 게이트 통과 전 다음 금지"

작성일: 2026-07-20
상황: 최근 13일간 17커밋 중 다수가 "수비 AI 수정" 반복 — 커버 미스/공 궤적/선수 이동이 서로 얽혀 고칠수록 꼬이는 루프. **유저 결정: 안무·수비AI 계층 전면 초기화.** 필드 이미지+앵커, 스프라이트, 페이싱, HUD, 엔진(PA 이벤트)은 유지한다.

## 0. 이 루프가 반복되는 진짜 이유

모션과 커버 로직을 **브라우저에서 눈으로 보고 고치는 방식**으로 개발하고 있어서다. 움직임 버그는 서로 연쇄되고(공 궤적이 틀리면 수비 도착이 틀리고 커버가 틀림), 눈검수는 한 번에 전부를 보므로 원인 분리가 안 된다. 리셋의 핵심은 그림이 아니라 **개발 순서와 검증 방법**을 바꾸는 것:

1. **모션 커널 = 순수 함수 모듈** (`src/gamecast2/kernel/`): 입력(PA 이벤트+앵커) → 출력(모든 개체의 시간별 위치 트랙). DOM/Phaser/캔버스 접근 금지.
2. **Node 단위테스트가 1차 검증** — 브라우저를 열기 전에 수학으로 통과. `tools/verify_motion.mjs` 신설(verify 3종에 추가, 브라우저 불필요).
3. **커버는 AI가 아니라 야구 규칙 룩업 테이블** — 추론·휴리스틱 금지. 실제 야구의 수비 시프트/커버/백업은 상황별로 정해져 있다(6-4-3, 외야 중계, 베이스 커버). 표로 박는다.
4. **한 마일스톤 = 한 종류의 움직임.** 게이트(단위테스트+버스트 캡처) 통과 전 다음 단계 착수 금지. "겸사겸사 수정" 금지 — 이게 17커밋 루프의 원인.

## 1. 마일스톤 (M0~M7)

| 단계 | 움직이는 것 | 내용 | 게이트 (Node 테스트 + 버스트 1회) |
| --- | --- | --- | --- |
| **M0** | 없음 | 기존 안무/수비AI 코드 격리(삭제 아닌 미사용 처리), `kernel/` 골격 + `compileTracks(event, anchors)` 시그니처 + verify_motion.mjs 빈 러너 | verify 3종 PASS 유지 |
| **M1** | **공만** | 투구 직선(마운드→홈 0.4s) + 타구 포물선: 이벤트의 `battedBallType`+`fieldingPosition`으로 낙하점 존 결정(deterministic), 체공시간 = 타구질별 상수 | 이벤트 30종 재생 시 낙하점이 담당 포지션 존 안(테스트) · 화면에서 공 항상 보임 |
| **M2** | +타자주자 | home→1B(→진루) 고정 속도 프로필, 아웃컴별 도달 베이스 초과 금지 | 경로가 베이스패스 앵커 위(오차 ≤4px) · 1루 방향 · outcome별 최종 위치 정확 |
| **M3** | +담당 수비수 1명 | 낙하점 요격: 도착시간 = 공 도착시간 동기, run→catch. **나머지 8명 동결** | 요격 지점 = 낙하점(테스트) · 정지 수비수 드리프트 0 |
| **M4** | 없음(데이터) | **커버 테이블**: (타구질×방향) → {fielder, 베이스 커버 맵 {1B:누구,2B:,3B:,home:}, 백업 1명}. 실제 야구 규칙으로 상황 열거(예: 유격 땅볼 → SS 처리, 2B가 2루, 1B가 1루, LF 백업) | 전 조합에 대해: 처리자≠커버어, 포스 베이스 전부 커버, 이동자 ≤4명(테스트만, 화면 무변화) |
| **M5** | +커버 이동/송구 | M4 표대로 커버어 이동 + 송구 체인(fielder→베이스, 병살은 2단), 공 리시브 타이밍 동기 | 송구 도착=리시버 도착(테스트) · 병살 6-4-3 시퀀스 버스트 확인 |
| **M6** | +기존 주자 | basesBefore→basesAfter 경로, 플라이 태그업(포구 후 출발), 득점 홈인 | 주자 수·최종 베이스가 이벤트 데이터와 일치 |
| **M7** | 마감 | 배지 후행 동기, 카메라(홈런만), 여운. 구 안무 코드 삭제 | 기존 verify:browser 랩 검사 전부 PASS |

## 2. 커널 계약 (전 단계 공통)

- `compileTracks(event, anchors) → { durationMs, entities: [{id, kind, keyframes:[{t, x, y, anim}]}] }` — **시간 t는 0..1 정규화**, 렌더러는 보간 재생만.
- 결정론: 같은 이벤트 = 같은 트랙(hashParts만 허용). 모든 좌표는 앵커 보간 — 손좌표 금지(기존 원칙).
- 렌더러 바인딩은 각 마일스톤 마지막에 1회 — 커널 수정과 렌더 수정을 같은 커밋에 섞지 않는다.
- verify_motion.mjs: 커널만 import해서 대표 이벤트 30종(아웃컴×타구질 조합) 자동 검사. **이 파일이 그린이 아니면 브라우저를 열지 않는다.**

## 3. 금지 목록 (루프 재발 방지)

1. "화면 보고 좌표 몇 px 조정" 커밋 금지 — 좌표는 앵커/테이블/공식에서만 나온다.
2. 커버 판단에 거리계산·스코어링 같은 휴리스틱 금지 — M4 표에 없는 상황이면 표를 확장한다.
3. 마일스톤 건너뛰기·병합 금지. 커밋 제목에 `[M3]`처럼 단계 표기.
4. 버스트 캡처는 게이트 확인용 1회 — 개발 중 반복 눈검수로 방향 잡지 말 것(테스트 먼저).

## 4. 검증

```powershell
node tools/verify_motion.mjs     # 커널 단위테스트 (신설, 게이트 1차)
npm run verify && npm run verify:browser
node tools/capture_gamecast_shots.mjs burst   # 게이트 2차(눈검수 1회)
```
