## 시스템별 요구사항 정리

### 1. 실제 로스터 DB

출처: 2026-02-26, 2026-02-27 로스터 글.

- 목표 규모는 10개 구단 1,052명.
- 선수 기본 프로필: 이름, 소속, 포지션, 투타, 등번호, 생년월일, 신체, 출신교, 지명 정보, 병역, 외국인 여부.
- 계약/상태: 연봉, 계약 상태, FA 예정, 보류권/외국인 슬롯, 군 복무 상태.
- 기록: 2025 시즌 기준 타자/투수 기본 기록과 세부 기록.
- 향후 능력치 산정 재료: 타석/타수, 출루/장타, 홈런, 도루, 득점권, FIP/BABIP/WAR/WPA 같은 추가 지표 후보.
- 우리 현재 상태: 531명 + 2026 공식 기록. 다음 확장은 2025 기준 전체 등록/육성/퓨처스까지 넓히는 데이터 수집 작업이다.

### 2. 선수 능력치

출처: 2026-03-08, 2026-03-10 능력치 글.

- 기본 체계: FM식 내부 1-20.
- 표시 체계: OVR/POT 0-200.
- 현재 기준 47개 능력치: 공통, 투수 전용, 타자 전용으로 분리.
- 확장 방향: 이후 60개까지 확장 가능. 후보는 수비 포지션별 세분화, 구종/구속/무브먼트, 좌우 split, 멘탈/리더십/성장성 세분화.
- 잠재력: 선수 전체 POT뿐 아니라 개별 능력치별 성장 여지를 둔다.
- 우리 현재 상태: 47개 능력치 구현 완료. 실제 split/수비 상세가 없으면 공식 기록 + 보수 fallback으로 처리 중.

### 3. 시즌 시뮬레이션

출처: 2026-03-08, 2026-03-09.

- 목표: 10개 팀, 팀당 144경기.
- 이상적 엔진: 경기 단위가 아니라 타석 단위 결과 생성.
- 반영 요소: 선수 능력치, 좌우 상성, 컨디션, 피로, 사기, 투수 팔 상태, 부상, 경기장, 날씨, 관중.
- 장기적으로 box score, 선수 누적 기록, 순위표, 기록 경쟁이 필요하다.
- 우리 현재 상태: 2026-03-01 프리시즌부터 개막일까지 하루씩 진행하고, 정규시즌은 일자별 5경기와 타석 이벤트 v1, line score, 선수 누적 기록을 만든다. UI에서는 전체 시즌 즉시 진행 버튼을 노출하지 않는다.

### 4. 밸런스 튜닝

출처: 2026-03-09.

- 기준점은 2024 KBO 평균.
- 검증 대상: 타율, 출루율, 장타율, ERA, 경기당 득점, 홈런, 도루, 볼넷, 삼진, 부상 발생 수.
- 튜닝 방식: 한 시즌 자동 시뮬레이션을 반복하고 리그 평균이 목표값에 수렴하는지 확인.

### 9. FA/외국인 시장

출처: 2026-03-21.

- FA 등급: A/B/C, 보상, 35세 이상 예외 등 KBO 규정 반영.
- 계약 구조: 계약금, 연봉, 인센티브, 옵션, 옵트아웃, 트레이드 거부권.

- AI GM도 시장에 참여하며 구단 성향에 따라 공격성/가성비 선호가 달라진다.

v1 반영 기준:

- 엔진 export 구현: `initializeFreeAgency(state)`, `commitFreeAgentSigning(state, offer?)`, `commitForeignPlayerSigning(state, offer?)`.

- FA 후보는 실제 로스터 DB의 국내 선수 중 `faStatus`와 `compensationGrade` 기반 후보만 사용한다.
- FA 계약 시 선수 `teamId`, `contract`, `faStatus`와 보상 ledger를 갱신한다.
- 보상 룰 v1은 A=20인 보호+전년도 연봉 200% 또는 300% 현금, B=25인 보호+100% 또는 200% 현금, C=150% 현금/보상선수 없음이다.
- 외국인 시장은 검증된 실명이 아니므로 `FGN-YYYY-###` 코드형 30명/5티어 후보로 만들고, 계약 시 roster에 가짜 이름을 넣지 않는다.
- 외국인 계약 결과는 보류권/활성화 대기 ledger로만 남기며, 실제 실명/국적/이력 데이터가 확보될 때 roster 반영을 별도 처리한다.

## 구현 의존성

1. 로스터/계약/기록 DB가 먼저다.
2. 능력치 산정은 DB 위에 얹는다.
3. 타석 단위 경기 엔진은 능력치와 실제 리그 평균 목표가 필요하다.
4. 선수 누적 기록이 있어야 포스트시즌/시상식/FA 가치/트레이드 가치가 의미 있어진다.
5. 드래프트/FA/트레이드/스카우트는 같은 선수 가치 모델을 공유하되, 서로 원본 데이터를 덮어쓰면 안 된다.
6. 뉴스/여론은 마지막에 붙이는 편이 안전하다. 핵심 로직이 불안정할 때 붙이면 placeholder와 가짜 이벤트가 확산된다.
