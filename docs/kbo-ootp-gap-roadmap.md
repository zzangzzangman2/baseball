# KBO Native Sim Depth Roadmap

## 방향

OOTP를 그대로 복제하지 않는다. 이 프로젝트는 KBO 10개 구단, 실제 로스터, KBO 규정, 귀여운 도트 중계, 결정론 시뮬레이션을 강점으로 삼는다. 격차를 줄일 때도 MLB식 거대 유니버스보다 KBO에서 체감되는 운영 요소를 먼저 넣는다.

## 이미 반영된 것

- 좌우 상성: 타석 판정이 `contactL/contactR`, `powerL/powerR`, 투수 `vsLHB/vsRHB`를 실제 사용한다.
- KBO 구장 성향: 홈팀 구장별 park factor가 경기 환경에 합성된다.
- 포지션 그룹 수비: 포수/내야/외야 수비 품질이 타구 유형별 안타/실책/병살 확률에 반영된다.
- 병살/실책: `plateAppearanceEvents`, 박스스코어, 시즌 `fielding.errors`, 타자 `reachedOnErrors/groundedDoublePlays`에 기록된다.
- 균형 검증: 2024 KBO 평균 기준 AVG/ERA/HR 검증을 유지한다.

## KBO식 우선순위

1. 인게임 감독 AI와 유저 라인업/로테이션 편집
   - KBO는 불펜 의존, 좌우 대타, 포수/수비 교체 체감이 크다.
   - 자동 위임과 직접 편집을 둘 다 제공한다.

2. 연중 로스터 생태계
   - 1군/퓨처스 엔트리 이동, 확대 엔트리, 부상자 말소, 외국인 교체, 군보류를 로스터 트랜잭션으로 통일한다.

3. FA/트레이드 시장의 능동 AI
   - 10개 구단이 팀 니즈, 예산, 보상등급, 외국인 슬롯, 순위 경쟁 상태로 움직인다.
   - 마감일 전후로 CPU-CPU 트레이드 빈도를 올린다.

4. 스카우트 불확실성
   - 유망주뿐 아니라 1군 선수도 관측치와 실제 능력치를 분리한다.
   - 스카우트 성향/피로/지식이 보고서 신뢰도에 영향을 준다.

5. 타석 엔진 v2
   - 볼카운트, 투구수, 구종, 포수 프레이밍, 도루/견제/번트/히트앤런을 단계적으로 추가한다.
   - 단, 매 단계마다 `verify:balance`가 2024 KBO 기준을 벗어나지 않아야 한다.

6. KBO 유니버스 확대
   - 531명 실명 로스터를 기준으로, 퓨처스/육성/드래프트/외국인 후보를 검증 가능한 범위에서 확장한다.
   - 이름 없는 저능력 placeholder 대량 생성은 지양한다.

## 구현 원칙

- 같은 입력은 같은 결과가 나와야 한다. 새 확률도 `rollUnit`/`hashParts` 기반으로 만든다.
- 실명/포지션/구단 정보는 임의로 만들지 않는다.
- 구장/규정처럼 매년 바뀌는 값은 상수에 source/version을 남긴다.
- UI보다 엔진이 먼저 진실을 들고, UI는 그 이벤트를 재생한다.
