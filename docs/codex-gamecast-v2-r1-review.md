# Codex 검수: 게임캐스트 v2 R1 리뷰 — 필드 합격, 앵커 4건 수정, R2(선수) 진행

작성일: 2026-07-07
방법: `?engine=v2&holds=0` 버스트 16프레임 + `?debug=anchors` 오버레이 캡처 + 콘솔 에러 수집(0건).

## 1. 합격 판정

- **필드 이미지**: 방송 앵글/모잉 스트라이프/흙 다이아몬드·베이스패스/원형 마운드/배터박스/워닝트랙/관중석 원근 — 전부 정상. 구 렌더러와 비교 불가 수준으로 좋아짐. "이미지가 기하다" 접근 검증 완료.
- **흰선(파울라인)**: 이미지에 구워진 파울라인이 홈→폴에서 정확히 종료, 이중선/관중석 관통 없음. 화면의 추가 흰 경로선은 `?debug=anchors` 전용 오버레이(정상 — 일반 모드에선 안 보임 확인).
- **폴/전광판 앵커**: leftPole/rightPole 담장 교차점 정확, scoreboardTL 정확.
- 앵커 디버그 오버레이 자체가 잘 작동 — 이 검수도 그걸로 했다.

## 2. 치명: 선수가 아예 렌더되지 않는다 (R2 미구현 상태)

- 홈런 진행 중("홈런!" 콜아웃, 점보 HR 1-0)인데 **투수·타자·포수·수비·심판·공 전부 0명**. 콘솔 에러 0건 = 로드 실패가 아니라 **스프라이트 배치 계층(R2) 자체가 아직 없음**.
- v2는 현재 R1(필드+앵커+오버레이)까지만이다. **R2(수비 9명 배치+아이들) → R3(투구) → R4(타구/주루)를 블루프린트 순서대로 진행할 것.** 그 전까지 v2를 기본 엔진으로 켜지 말 것(무인 구장 경기가 나감).

## 3. 앵커 수정 4건 (`field-jamsil-day.anchors.json` 외 3장 공통 — 재마킹 후 빌더 재실행)

| 앵커 | 문제 | 수정 |
| --- | --- | --- |
| `first`/`third` (베이스) | 베이스 백이 아니라 **파울라인 바깥쪽**으로 밀려 있음 — 이대로면 주자가 베이스 밖을 돎 | 흙 다이아몬드 코너의 베이스 백 중심으로 |
| `1B`/`3B` (수비 위치) | 베이스 백에 붙어 있음 | 베이스에서 잔디 쪽으로 1.5보 뒤(수비 정위치) |
| **`LF`/`CF`/`RF`** | **심하게 얕음** — 내야 흙 경계 바로 뒤에 서게 됨(CF가 2루 뒤 몇 걸음) | 내야 경계~담장의 55~65% 지점(CF 최심) |
| `second` | 베이스 백 대비 미세 상향 의심 | 백 중심 재확인 |

- 수정 후 `?debug=anchors` 스크린샷을 커밋에 첨부(관례). 주루 경로선이 흙 베이스패스 위에 정확히 얹히면 통과.

## 4. 잔여 (블루프린트 그대로)

- 전광판이 빈 검정 — R5에서 코드 렌더(이닝/스코어/타자명 블록레터).
- R2 통과 기준 재확인: 수비 9명이 앵커에 서고 깊이 scale 적용(외야 축소), 버스트 캡처로 확인.
